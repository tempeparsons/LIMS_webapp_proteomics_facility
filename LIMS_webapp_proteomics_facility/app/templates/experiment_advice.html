Estimating protein concentration:

A approximate protein concentration enables efficient digestion and injection of optimal volumes on to instruments, which means better data and fewer re-injections.

If you aren't regularly performing protein concentration assays in your lab, then this 2003 publication and this guide from ThermoFisher are useful resources. Assays can be usually be adapted for microplates or nanodrops where sample preservation is critical.

If you're sending in peptides, give concentrations from the last known measurement (ug/ul) before digestion.
If you're cutting a gel band, assume a 20ul resuspension volume.

Concentrations need only be very approximate:

  • For undetectable concentrations, report '0.1 ug/ul' and fill out the 'extra notes' section as appropriate.
  • Low-good concentration ~ 0.5 ug/ul
  • Optimal is 1.0 ug/ul
  • Loads would be > 1.5 ug/ul

    Estimating the number of injections:

    Instrument time in charged per injection. If we are fractionating one of your samples into 12 fractions, you'll need 12 injections. Likewise if we're cutting 4 bands from your gel, you'll need 4 injections. This column can't be blank, so take your best guess. This is simply for initialising file-name lists - you won't be over-charged if you guess too high!

    The standard data analysis package:

    If you want your data run through some commercial or open-source software that we use in the facility (see Home > Services > Data Analysis Service for a full list), then you most likely want to select the standard data analysis package. This includes anything from 15 minutes to 2 hrs of analysis using the open source.freeware or proprietary pieces of software used by the facility. If you require any additonal analysis (e.g. seaching against a different database, changes to graphical presentation) a new 'data_analysis_only' request must be submitted.

    Requesting data analysis only:

    Data analysis can be requested without submitting samples. You will still need a unique experiment code for this request, so we can track it. Choose 'Data analysis only, no samples' from the dropdown menu and then hit 'Submit all'. Typically, analysis-only requests are made for when specific python or R scripts need to be written, a user-specific piece of software is required, or for when adjustments are required to a pre-agreed standard analysis output. We can discuss these options with you as per your request.

    Choosing experiment and sample codes:

    Every user-request made via the sample submission form gets associated with an experiment code. Every code for an individual user must be unique, otherwise we can't retrieve it from the database. If a single experiment is split over two sample submissions, two unique codes will be required. An experiment code consists of up to 6 alphanumeric characters excluding underscores. There is no need to include group leader or researcher intials in the experiment code as these are automatically prefixed on to each instrument injection code. Experiment and sample codes are chosen by the user, though we recommmened a simple and concise format

    File name patterns:

    Each injection on to the instruments is uniquely named. The file name pattern is: GroupInitials_ResearcherInitials_ExptCode_SampleCode_FractionCode_RepeatInjectionCode. The fraction code refers to how many injections you estimated that one sample would generate. A repeat injection code is only included if the sample needs to be re-injected with, for example, a different method or volume.

    Column headers in Sample Submission Form:

    Experiment code: A single experiment, consisting of multiple samples, is given a short identifier that is easily written on a tube. Each user's code must be unique. Different users may use the same code.
    Sample codes: Each sample within one experiment receives a short code, so that "expt_code, sample_code" are easily written on a tube. Sample codes must be unique within any one experiment, but may be reused between experiments.
    Protein concentration: required for every sample. See above advice. If the facility is measuring the concentration, put '0.1'.
    Number of injections: see above description.
    Run gel: Tick this box if the facility is running your gels for you. We run pre-cast 10% mini-gels as standard but other options are possible.
    In-gel digest: Tick these boxes if the facility is cutting bands and doing an in-gel digest for you
    S-trap: Method for cleaning up SDS-gel loading buffer without running a gel, permitting gel-free digest.
    In-solution or on-bead digest: Tick this box if you are providing proteins suspended in a Trypsin-compatible buffer or we are performing an on-bead digest for you. Please contact us for help or advice on preparing protein solutions.
    LysC digest: LysC is appreciably more expensive than all our other enzymes, so incurs an additional charge (see pricing page).
    PO4-enrichment: Phospho-peptide enrichment. Although most groups perform their own enrichments, we can do this in house. We typically use the Zr-IMAC HP (MagReSyn) enrichment kit.
    labelling: Suitable for experiments where very high accuracy relative quantitation is required. Labelled peptides are run on our Fusion Lumos. Users can choose from in house-aliquotted TMT labels (cheapest option), supplying their own kits or purchasing them through the facility.
    Fractionation: Recommended for complex samples requiring a complete survey of proteins present. We typically perform reverse-phase HPLC, collecting 12 fractions, but can cater for specific requests.
    Intact mass: Check this box if you are sending us a purified protein solution for intact mass-analysis that is ready to resuspend or load directly on an instrument.
